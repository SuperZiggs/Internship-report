During my internship, I participated in some events. Each one was a memorable experience that provided new, interesting, and useful knowledge, along with gifts and wonderful moments.

### [Event 1](4.1-Event1/)  
&emsp;**Event Name:** AWS re:Invent 2025 Recap - Vietnam Edition  

&emsp;**Date & Time:** January 27, 2026  

&emsp;**Location:** AWS Vietnam Office (Floor 26 & 36), Ho Chi Minh City  

&emsp;**Role:** Attendee

&emsp;**Description:** This event delivered a recap of the major announcements from AWS re:Invent 2025, organized for the Vietnamese developer and cloud community shortly before Tết 2026. The sessions highlighted key innovations across the AWS ecosystem, particularly in Generative AI, machine learning platforms, and modern data infrastructure. Participants were introduced to technologies such as Amazon Bedrock and the Nova model family, along with Bedrock Agents that allow AI systems to execute complex workflows using orchestration, memory management, and guardrail policies. The event also covered updates to SageMaker Unified Studio, improvements in Amazon S3 including S3 Tables and S3 Vector, and new developments in OpenSearch that support agent-based search systems. Additional discussions explored advanced Retrieval-Augmented Generation (RAG) with multimodal embeddings and AI infrastructure tools such as SageMaker HyperPod and MLflow for large-scale machine learning workflows.  

&emsp;**Outcomes / Value Gained:** Through this event, I gained insights into emerging trends such as Agentic AI and multimodal RAG systems within the AWS ecosystem. I also learned about more cost-efficient approaches to vector storage using S3 Vector and explored how Bedrock Agents could potentially be applied to conversational or automation features in cloud-based applications. Furthermore, the event provided opportunities to interact with AWS Solution Architects and learn best practices related to serverless architecture and scalable cloud system design.

### [Event 2](4.2-Event2/)

&emsp;**Date & Time:** 09:00, March 14, 2026  

&emsp;**Location:** 26th Floor, Bitexco Tower, 02 Hai Trieu Street, Saigon Ward, Ho Chi Minh City  

&emsp;**Role:** Attendee

Industry speakers introduced the concept of AI agents capable of multi-step reasoning and interacting with external tools. The session also discussed best practices for prompt engineering to optimize large language model outputs and reduce operational costs. These insights are valuable for my future work in AI system development and cloud-based architectures.

### [Event 3](4.3-Event3/)  
&emsp;**Event Name:** Cloud Mastery 2026 #2 - Kubernetes, IaC, and Elixir in DevOps  

&emsp;**Date & Time:** 09:00, April 04, 2026  

&emsp;**Location:** FPT University HCM campus  

&emsp;**Role:** Attendee  

&emsp;**Description:** This event combined three technical sessions: Kubernetes architecture and operations, Elixir/OTP for highly concurrent and fault-tolerant systems, and Infrastructure as Code practices on AWS with CloudFormation, AWS CDK, and Terraform. The Kubernetes track covered control plane and worker node components, core objects, and practical tooling such as Helm and K9s. The Elixir track emphasized BEAM concurrency, supervision, and the "Let It Crash" philosophy for resilient platforms. The IaC track compared AWS-native and multi-cloud approaches, including template/stack concepts, CDK constructs, Terraform workflows, and state management.

&emsp;**Outcomes / Value Gained:** Through this event, I gained a more connected cloud engineering perspective across orchestration, runtime reliability, and infrastructure automation. I also developed a clearer practical roadmap: start with local Kubernetes practice, apply IaC consistently for deployments, and evaluate resilient runtime technologies such as Elixir for high-throughput services.

### [Event 4](4.4-Event4/)
&emsp;**Event Name:** Cloud Mastery 2026 #3 - Networking, Security, and IAM on AWS  

&emsp;**Time:** 09:00, April 11, 2026  

&emsp;**Location:** FPT University, Ho Chi Minh City Campus  

&emsp;**Role:** Attendee  

&emsp;**Event Description:**  
The event consisted of three technical sessions focusing on networking, security, and identity management on AWS. The networking session covered VPC design, subnets, Internet Gateway, route tables, and strategies to control traffic flow and minimize blast radius. The IAM session highlighted best practices such as the principle of least privilege, use of IAM roles, MFA, and multi-account management with AWS IAM Identity Center (SSO), along with advanced controls like SCPs and Permission Boundaries. The security session introduced key AWS protection services including AWS WAF, AWS Shield, AWS Network Firewall, and Firewall Manager to defend against DDoS attacks and common web exploits.  

&emsp;**Outcomes and Value Gained:**  
Through this event, I gained a clearer understanding of designing secure network architectures using VPC and tiered subnets, as well as implementing multi-layered security from network to application level. I also learned how to manage identities and permissions in a multi-account environment and developed a stronger awareness of applying a zero-trust security model in cloud systems.
